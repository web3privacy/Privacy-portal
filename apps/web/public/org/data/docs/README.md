# Docs content (org web)

This folder is served at **/data/docs** and powers the integrated Docs section on the org web (/docs).

## Structure

- **sidebar.json** — Left navigation (categories and links). Edit to add/remove pages or change order.
- **manifest.json** — Generated by the fetch script; maps URL slug → file path. Used to resolve links to the correct file (e.g. `about-us/culture` → `about-us/Culture.md`).
- **Portal & Org Web** — Local category under `portal-and-org-web/` (Overview, Data structure, Running locally, Testing, Dependencies). These are maintained here only until the full portal release.

All other content is synced from [web3privacy/docs](https://github.com/web3privacy/docs).

## Updating from upstream (content + images)

From the **repository root**:

```bash
npm run docs:fetch
```

This fetches the latest `.md`/`.mdx` and **all assets** (images, SVGs) from `web3privacy/docs` main into this folder and `assets/`. Images are mirrored locally so the app serves them from `/data/docs/assets/` without calling GitHub at runtime.

When you run **`npm run dev`** in `w3pn-org-web`, the same fetch runs first so docs and images are always up to date when you start the app.

## Local edits

You can edit any file under this folder. Changes are **not** pushed back to the GitHub docs repo until the full portal is released; the layout and fetch script are prepared for future sync if needed.

## MDX vs. Markdown

Content from web3privacy/docs is in **MDX** (Astro/Starlight) and uses components like `<LinkCard>`, `<CardGrid>`, and `import` statements. The org web renders docs with **react-markdown**. A sanitizer (`src/utils/sanitizeMdxForMarkdown.js`) runs before render and:

- Strips YAML frontmatter and `import ... from '...'` lines
- Converts `<LinkCard title="..." href="..." />` into markdown list links
- Removes `<CardGrid>` wrappers
- Strips any remaining JSX-style component tags

So readers never see raw `<LinkCard>` or metadata; internal hrefs are rewritten to `/docs/...` where needed.

## Adding a new page

1. Add the `.md` (or `.mdx`) file in the right folder (e.g. `portal-and-org-web/my-page.md`).
2. Add an entry in **sidebar.json** in the right category, e.g. `{ "label": "My Page", "link": "portal-and-org-web/my-page" }`.
3. For fetched docs, the manifest is regenerated by `docs:fetch`; for local-only pages the app resolves `slug.md` automatically.
